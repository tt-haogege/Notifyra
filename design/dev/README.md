# Notifyra 开发手册索引

本文档用于帮助在新对话、换模型、或中断后恢复开发时，快速找到必须阅读的资料与执行顺序。

## 1. 首次接手必读顺序

1. `design/RequirementDesign.md`
2. `design/ProductDesign.md`
3. `design/TechnicalDesign.md`
4. `design/DevelopmentPlan.md`
5. `design/dev/ProjectOnboarding.md`
6. `design/dev/DevelopmentWorkflow.md`
7. `design/dev/ExecutionChecklist.md`
8. 与当前任务直接相关的专项文档

## 2. 本目录文档说明

- `DevelopmentWorkflow.md`
  - 总体开发流程、TDD、review、收尾规范
- `ProjectOnboarding.md`
  - 新模型 / 新对话快速接手指南
- `ExecutionChecklist.md`
  - 从需求到落地的执行清单
- `BackendDevelopmentGuide.md`
  - backend 开发约定与常用模式
- `FrontendDevelopmentGuide.md`
  - frontend 开发约定与常用模式
- `ReviewChecklist.md`
  - review 时该检查什么、不该过度放大的点
- `TestingGuidelines.md`
  - 测试编写、验证顺序、TDD 落地方式
- `GitWorkflow.md`
  - git、提交、PR 与收尾约定
- `DocumentationMaintenance.md`
  - 文档维护责任与更新规则

## 3. 使用规则

- 任何模型开始开发前，都应先看本目录索引和相关文档。
- 如果开发流程、规范、阶段边界、review 规则发生变化，必须同步更新本目录文档。
- 本目录文档优先记录“如何开发”和“如何接手”，不重复记录产品需求本身。

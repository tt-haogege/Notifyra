# Notifyra Git 流程规范

## 1. 总原则

- 除非用户明确要求，否则不要主动创建 commit
- 除非用户明确要求，否则不要主动 push
- 不做危险 git 操作
- 不强推主分支

---

## 2. 提交前检查

提交前确认：
- 只包含本次改动相关文件
- 不包含敏感信息
- 不包含实验代码
- 测试与 build 状态清楚

---

## 3. commit 规范

commit message 应该：
- 简洁
- 准确反映改动目的
- 优先写清为什么，不只是写做了什么

---

## 4. 分支与工作树

如果当前使用 worktree：
- 在当前 worktree 内完成开发与验证
- 不随意切回原仓库目录
- 收尾时按用户意愿决定保留、合并或清理

---

## 5. 风险操作

以下操作默认要谨慎：
- 删除分支
- 覆盖未提交改动
- reset --hard
- force push
- 修改共享配置

如非必要，不应主动执行。
